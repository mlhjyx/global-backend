# 产品体验原则

> 文档 ID：`FE-GLOBAL-002`
> 层级：`L2 / Normative candidate`
> 生命周期：`ACTIVE_INPUT`
> 评审状态：`READY_FOR_GATE_4_REVIEW`
> 内容 Owner：`OWN-DESIGN`
> 主要 Actor：`ACT-FE-002`；适用于全部 Capability

## 1. 原则

| Principle ID | 原则 | 可验证行为 |
|---|---|---|
| `EXP-FE-001` | 先展示用户结果，再展示工具 | 首屏回答当前状态、影响和下一步；不以模型、Provider 或工作流名组织主导航 |
| `EXP-FE-002` | 结构化对象是产品真相 | AI、通知、Today、搜索和审批都深链到 canonical object；聊天记录和卡片不拥有业务状态 |
| `EXP-FE-003` | Evidence 先于自信 | 公开事实、推荐和审批可查看来源、时间、范围和未知项；不以“AI 生成”作为证据 |
| `EXP-FE-004` | 自动化逐级放权 | 草拟、建议、内部变更、外部动作分级；发布、发送、导出、删除等高影响动作受授权和人工门约束 |
| `EXP-FE-005` | 失败时保住已有价值 | 新任务失败、取消或降级时明确旧结果是否仍可用；不把候选失败显示为整个对象失败 |
| `EXP-FE-006` | 状态、成本和限制必须诚实 | `partial/degraded/stale/unknown/skipped` 不并入成功；estimate 不冒充实际，preview 不冒充 publish |
| `EXP-FE-007` | Workspace、身份和权限上下文始终可见 | 关键动作前可确认 Workspace、对象、Owner、权限范围和影响；客户端不自建授权真值 |
| `EXP-FE-008` | 为高频企业工作保持密度和渐进披露 | 摘要可扫描，细节可展开，批量任务可操作；避免营销式 Hero、装饰卡片和低信息密度 |
| `EXP-FE-009` | 全球化和可访问性默认开启 | 键盘、屏幕阅读器、缩放、长文本、时区、locale 和降级在设计阶段覆盖，不作为发布后补丁 |
| `EXP-FE-010` | 衡量用户价值，不奖励系统活动 | 看可用结果、恢复、事实质量和业务结果；页面数、Build success、AI 调用数不能单独代表成功 |
| `EXP-FE-011` | 系统一致但不抹平业务差异 | Shell、权限、状态、审批和 Evidence 复用；模块只增加特有对象/动作/状态，不复制基础模式 |
| `EXP-FE-012` | 可退出、可追溯、可接手 | 有稳定 URL、历史、导出/保留政策、correlation ID、人工兜底和 Owner；不能把用户锁在一次会话里 |

## 2. 关键设计判断

### 2.1 “今日”不是算法首页

Today 只聚合有来源的任务、审批、异常、机会和恢复项。每张卡必须说明为何出现、属于哪个 Workspace/对象、截止/新鲜度、可执行动作以及 dismiss/snooze 规则；没有可解释排序时按明确规则展示。

### 2.2 AI 不是另一个产品层

AI 可以帮助表达目标、解释状态、生成草稿和启动有界任务，但不能绕过对象、权限、Evidence、预算或 Approval。可复用结果必须写入明确对象；一次聊天结束后仍能被找到、编辑、审批和审计。

### 2.3 统一不等于“一张万能页面”

全局模式统一的是状态、动作、内容和证据语义。Company、Site、Build、Opportunity 等仍有自己的 canonical 页面和生命周期；聚合中心只提供跨域视图，不吞并对象 ownership。

## 3. 体验审查问题

每个页面/流程进入设计评审前至少回答：

1. 用户此刻要完成什么结果，第一屏能否看懂？
2. 当前对象、Workspace、Owner、状态和 allowed actions 来自哪里？
3. 空、无权、未配置、等待、部分成功、失败、取消、stale、离线如何表现？
4. 系统保留了什么，用户下一步能做什么，人工兜底是谁？
5. AI 输出哪些是事实、推断、建议、草稿；Evidence 和批准在哪里？
6. 键盘、焦点、屏幕阅读器、缩放、长文本和移动端能否完成关键任务？
7. 埋点衡量的是用户结果还是点击/调用活动？
8. 有无稳定 Page/Capability/Scenario/Asset/Copy ID 和 Owner？

## 4. 反模式

- 用渐变 Hero、巨型数字或“AI 驱动”掩盖空数据、未知状态或未接合同。
- 每个模块各写一套 Toast、403、审批卡、成本标签或长任务轮询。
- 根据前端角色字符串显示危险动作，同时服务端合同不返回 allowed action。
- 把灰掉按钮当权限说明，把原始异常堆栈当错误文案，把 loading spinner 当长任务体验。
- 自动接受 Claim、静默 fallback、吞掉失败、覆盖旧结果或将 `unknown` 计入 success。
- 以“所有功能都在导航中”表示产品完整，或以隐藏未建能力掩盖产品地图。
